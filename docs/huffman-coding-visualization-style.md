# HuffmanCoding.html 시각화 스타일 정리

이 문서는 `Algorithms_01/14_greedy_algorithm/HuffmanCoding.html`에 적용한 화면 스타일과 구성 기준을 정리합니다.

## 기본 방향

- 어두운 VS Code 계열 배경을 사용합니다.
- 전체 글꼴은 모노스페이스 계열을 사용해 코드, 노드 값, 힙 배열을 같은 리듬으로 읽을 수 있게 합니다.
- 설명은 짧은 단계 설명 박스에 집중시키고, 시각 정보는 패널 안에서 직접 확인하도록 구성합니다.
- `priority_queue`와 `Main Huffman Tree` 패널은 같은 크기와 높이를 유지합니다.

## 화면 구성

- 상단에는 제목, 입력 데이터 설명, 탭, 단계 카운터, 단계 설명을 둡니다.
- 가운데에는 두 개의 주요 패널을 같은 크기로 배치합니다.
  - `priority_queue Min-Heap / 빈도 기준 최소 힙`
  - `Main Huffman Tree / 전체 메인 트리`
- 하단에는 실행 상태 카드와 코드 패널을 배치합니다.
- 출력 전용 패널은 사용하지 않습니다.

## 패널 스타일

- 패널 배경은 `#252526`, 테두리는 `#3e3e42`를 사용합니다.
- 패널 모서리는 `6px` 반경을 사용합니다.
- 주요 시각화 패널은 CSS Grid의 2열 구조로 배치합니다.
- 두 패널의 높이는 `align-items: stretch`와 `flex: 1` SVG로 동일하게 맞춥니다.
- 모바일 폭에서는 패널을 세로로 쌓습니다.

## 노드와 색상 의미

- 모든 트리와 힙 노드는 같은 원 크기와 텍스트 크기를 사용합니다.
- `top/current`: 주황 계열
- `left / 0`: 파란 계열
- `right / 1`: 보라 계열
- `new/push`: 초록 계열
- `heap swap`: 연두 계열
- leaf node는 기본 회색 테두리를 유지합니다.

## 힙 표현

- `priority_queue`는 실제 내부 heap 배열을 완전 이진 트리 형태로 표시합니다.
- 아래에는 heap 배열 인덱스와 노드 값을 표 형태로 함께 표시합니다.
- 빈도가 작은 노드가 위로 올라가는 구조를 `Compare` 단계와 swap 단계로 보여줍니다.
- 모든 초기 노드 push가 끝난 뒤 `1차 우선순위큐 완성` 단계를 별도로 둡니다.

## 허프만 트리 표현

- `Main Huffman Tree`는 현재 heap에 남아 있는 루트들과 병합 중인 노드를 forest 형태로 표시합니다.
- 병합이 진행될수록 작은 트리들이 합쳐져 하나의 메인 트리로 수렴합니다.
- 왼쪽 간선은 `0`, 오른쪽 간선은 `1`로 표시합니다.
- 별도의 서브트리 패널은 두지 않고, 메인 트리에서 현재 선택 노드를 하이라이트합니다.

## 단계 구성

- 초기 heap 구성 단계는 `heap.push()`와 heap swap을 모두 보여줍니다.
- 병합 루프에서는 두 번의 pop을 한 단계로 묶어 `left/right 동시 선택`으로 보여줍니다.
- `new_data`, `new_freq`, `top 생성`, `top->left`, `top->right`, `heap.push(top)` 순서를 보여줍니다.
- `cout` 출력 단계와 `PrintCodes()` 출력 단계는 시각화에서 제외합니다.

## 코드 패널

- 코드 패널은 하단에 배치하고, 활성 라인을 노란 계열 왼쪽 보더로 표시합니다.
- 라인 번호를 표시합니다.
- 키워드, 타입, 변수, 숫자, 주석에 색상을 나눠 적용합니다.
- 단계 이동 시 코드 패널로 자동 스크롤하지 않습니다.

## 조작 방식

- 버튼 조작을 지원합니다.
  - `Home`
  - `이전`
  - `재생`
  - `다음`
  - `End`
- 키보드 조작을 지원합니다.
  - `ArrowLeft`: 이전 단계
  - `ArrowRight`: 다음 단계
  - `Home`: 첫 단계
  - `End`: 마지막 단계
  - `Space`: 재생/정지

## 반응형 기준

- 넓은 화면에서는 `priority_queue`와 `Main Huffman Tree`를 같은 행에 둡니다.
- 좁은 화면에서는 두 패널을 세로로 쌓습니다.
- 힙 배열은 가로 스크롤을 허용해 작은 화면에서도 셀 폭이 과도하게 줄어들지 않게 합니다.
