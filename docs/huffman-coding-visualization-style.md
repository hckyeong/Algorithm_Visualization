# 시각화 HTML 스타일 정리

이 문서는 현재 시각화 HTML, 특히 `Algorithms_01/15_minimum_spanning_tree/PrimMST.html`에 적용한 화면 스타일과 구성 기준을 정리합니다. 이후 알고리듬 시각화 파일을 만들거나 수정할 때 같은 기준을 우선 적용합니다.

## 기본 방향

- 어두운 VS Code 계열 배경을 사용합니다.
- 전체 글꼴은 모노스페이스 계열을 사용해 코드, 배열, 그래프 라벨, 출력 결과를 같은 리듬으로 읽을 수 있게 합니다.
- 단계 설명은 상단에 집중시키고, 중요한 실행 상태는 그래프, 배열, `adj`, 출력 패널에서 직접 확인하도록 구성합니다.
- 단계 이동 중 화면이 위아래로 흔들리지 않도록 고정 높이가 필요한 영역은 전체 단계 중 가장 큰 표시량을 기준으로 높이를 고정합니다.

## 화면 구성

- 상단에는 제목, 단계 카운터, 단계 설명, 현재 실행 코드, 범례를 둡니다.
- 가운데에는 그래프 패널과 상태 패널을 2열로 배치합니다.
  - 왼쪽: `Undirected Weighted Graph`
  - 오른쪽: `IndexMinPQ`, `key 배열`, `pre 배열`
- 그 아래에는 `adj` 결과 패널을 전체 폭으로 배치합니다.
- `adj` 결과 아래에는 `cout` 출력 결과 패널을 전체 폭으로 배치합니다.
- 마지막에는 VS Code 스타일 코드 패널과 단계 이동 컨트롤을 배치합니다.
- 좁은 화면에서는 그래프, 상태, `adj`, 출력 패널이 세로로 쌓이게 합니다.

## 패널 스타일

- 기본 패널 배경은 `#252526`, 테두리는 `#3e3e42`를 사용합니다.
- 코드와 출력처럼 콘솔 느낌이 필요한 내부 영역은 `#1e1e1e` 배경을 사용합니다.
- 패널 모서리는 `6px` 반경을 기준으로 합니다.
- `main-content`는 넓은 화면에서 CSS Grid를 사용합니다.
  - 1행: `graph state`
  - 2행: `adj adj`
  - 3행: `output output`
- 모바일 폭에서는 `flex-direction: column`으로 전환합니다.

## 그래프 하이라이트

- Prim MST 그래프 간선은 세 그룹으로만 구분합니다.
  - `전체 그래프 T`: 기본 간선, 회색
  - `신장 트리 F`: 현재까지 선택된 MST 간선, 초록색
  - `비교해야할 간선`: 현재 `Adj(u)`에서 검사 중인 간선, 주황색
- 범례도 위 세 그룹만 보여줍니다.
- 그래프 간선과 `adj` 간선 카드는 같은 색상 의미를 공유합니다.
- 최종 단계에서는 MST 간선 8개가 `F`로 표시되고 나머지 간선은 `T`로 남아야 합니다.

## 배열과 상태 패널

- `key 배열`과 `pre 배열`은 카드가 여러 줄로 내려가지 않도록 한 줄 배열로 표시합니다.
- 배열이 길어지면 패널 내부에서 가로 스크롤되게 하고, 전체 레이아웃 폭을 밀어내지 않습니다.
- 배열 셀은 정점 번호와 값을 위아래로 보여줍니다.
- `IndexMinPQ` 내용은 `min -> max` 순서의 chip 형태로 표시합니다.

## adj 패널

- `adj` 패널은 `vector<vector<DirectedEdge>>`의 결과를 정점별 행으로 보여줍니다.
- 각 행은 정점 번호, 화살표, 인접 간선 카드 목록으로 구성합니다.
- `AddBiEdge()` 때문에 무방향 간선 하나가 양쪽 방향 `DirectedEdge`로 저장된다는 점을 패널 설명에 명시합니다.
- 현재 비교 중인 간선, 현재까지 선택된 신장 트리 간선, 기본 그래프 간선은 그래프와 동일한 색상 기준으로 표시합니다.

## 출력 패널

- `cout` 출력이 알고리듬 이해에 필요하면 출력 전용 패널을 둡니다.
- 출력 패널은 `adj` 결과 아래에 전체 폭으로 배치합니다.
- 출력이 발생하는 순간은 독립 step으로 분리합니다.
- 출력 패널에는 현재 step까지 누적된 출력만 표시합니다.
- 새로 출력된 줄은 노란 계열 왼쪽 보더와 배경으로 강조합니다.
- 출력 패널 높이는 최종 출력 줄 수 기준으로 고정해 step 이동 중 레이아웃이 흔들리지 않게 합니다.

## 단계 실행 코드 표시

- 단계 설명 박스 아래에는 현재 step에서 실행 중인 코드 줄을 별도 블록으로 표시합니다.
- 실행 코드 블록은 라인 번호와 코드 내용을 함께 보여줍니다.
- 원본 코드의 들여쓰기 깊이를 반영해 종속 관계를 읽기 쉽게 합니다.
- 들여쓰기 가이드는 VS Code 스타일의 약한 세로선으로 표시합니다.
- step마다 실행 코드 줄 수가 달라도 화면이 위아래로 움직이지 않도록, 실행 코드 블록 높이는 전체 step 중 가장 긴 실행 코드 묶음의 높이로 고정합니다.
- 긴 코드 줄은 블록 내부에서 가로 스크롤되게 하고, 전체 페이지 레이아웃을 흔들지 않습니다.

## 코드 패널

- 코드 패널은 VS Code 편집기 블록처럼 보이게 합니다.
- 라인 번호를 표시합니다.
- 키워드, 타입, 함수, 변수, 숫자에 색상을 나눠 적용합니다.
- 활성 라인은 노란 계열 왼쪽 보더와 약한 배경으로 표시합니다.
- 들여쓰기 가이드는 코드의 중첩 구조와 가깝게 맞춥니다.
- 단계 이동 시 코드 패널로 자동 스크롤하지 않습니다.

## 단계 구성

- 알고리듬 상태 변화와 출력 발생은 가능하면 분리된 step으로 보여줍니다.
- Prim MST에서는 다음 이벤트를 별도 step으로 둘 수 있습니다.
  - `AddBiEdge()`로 `adj`를 구성하는 단계
  - `V : 9`처럼 초기 출력이 발생하는 단계
  - `key`, `pre`, `pq` 초기화 단계
  - `DelMin()`으로 정점이 MST에 들어오는 단계
  - 확정된 간선을 `cout`으로 출력하는 단계
  - `Adj(u)`의 간선을 비교하는 단계
  - `key/pre`가 갱신되는 단계
  - 최종 `cost_sum`을 출력하는 단계
- 각 step은 설명, 현재 실행 코드, 그래프/배열/출력 상태가 서로 같은 의미를 가리키게 유지합니다.

## 조작 방식

- 버튼 조작을 지원합니다.
  - `이전`
  - `처음`
  - `다음`
- 키보드 조작을 지원합니다.
  - `ArrowLeft`: 이전 단계
  - `ArrowRight`: 다음 단계
  - `Home`: 첫 단계
  - `End`: 마지막 단계
- 공유 viewer control 스크립트와 충돌하지 않게 합니다.

## 반응형 기준

- 넓은 화면에서는 그래프와 상태 패널을 같은 행에 둡니다.
- `adj`와 출력 패널은 전체 폭을 사용합니다.
- 좁은 화면에서는 모든 주요 패널을 세로로 쌓습니다.
- 배열처럼 폭이 필요한 UI는 화면 전체를 밀어내지 않고 패널 내부 가로 스크롤을 사용합니다.
- 모바일에서도 step 컨트롤은 터치 가능한 크기를 유지합니다.
